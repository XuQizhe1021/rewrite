## SEO/内容垃圾过滤器（v1.0）需求文档

### 目标
- 面向网页阅读的“内容净化与总结”扩展：提取正文、截断上下文、后台代理调用 AI（流式）、多种呈现方式与入口。
- 严格限定在 v1.0：v2.0 多模态仅保留接口与文档占位，不实现。

### 核心能力
#### 内容提取
- 从当前页面 DOM 中提取正文、标题、元数据。
- 优先使用 Readability（或等价实现）。
- 需要去除广告/导航/评论/脚本等噪音并尽量保留段落与列表结构。
- 兜底：Readability 失败时回退到 body 文本，并排除 nav/footer/aside/script/style。

#### 上下文截断
- 目标总 tokens 约 3000–4000。
- 优先保留：标题、首段、包含“结论/总结/结果/最后”等关键词段落。
- 若可用：用 gpt-tokenizer（或等价库）估算 token；不可用则降级为字符数 + 关键词启发式。

#### AI 调用（安全优先）
- Background Service Worker 代理请求，统一管理 API Key/模型/参数，前端不接触 Key。
- Key 存储于 chrome.storage.local。
- 严禁硬编码与日志泄露 Key。
- 支持流式响应：Background 读取 ReadableStream，分 chunk 透传到前端实时渲染。

#### 渲染与入口
- 支持多呈现：Side Panel（优先/推荐）、悬浮层（页面内）、Popup。
- 结果 Markdown 渲染。
- 交互：复制、重新生成、满意度反馈。
- 可选 TL;DR 顶部条：淡黄色背景，一句话总结（≤30字）。
- 右键菜单支持 3 种模式入口：通用总结、操作步骤、对比分析。

#### 配置与偏好
- Options 页面配置：API Key、模型选择、温度、默认总结风格（简洁/详细/极简）、是否自动触发（域名白名单）。
- 自动模式：保守实现（域名白名单 + 明确提示 + 可撤销），遵循最小权限。

### 安全与隐私约束
- 不在任何前端脚本/仓库中硬编码或打印 API Key。
- 默认仅在用户触发时使用 activeTab 获取当前页内容。
- 页面内容仅发送到用户配置的 API 端点，不经过开发者服务器。
- 错误处理：请求失败、流式中断、模型不可用、token 超限等均需明确可理解提示，并可重试。

### 质量与验证
- 纯函数单元测试：文本清洗、关键词筛选、段落打分、截断策略。
- 自检脚本：构建产物体积、权限清单、manifest 基本校验。
- 手工验收覆盖：新闻 TL;DR、教程步骤、评测对比表、大文本截断 + 流式渲染。

### v2.0（仅占位）
- 多模态/图片理解仅保留配置入口与接口类型占位，不做图片提取与上传、不调用视觉模型。
